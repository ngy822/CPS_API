源代码分析报告

MATLAB 空间对接与 4-PR 并联调姿模型分析

分析对象为 G:\CPS_API\matlab程序 下的两份 MATLAB 脚本和一个当前测量数据文件。 这套程序的核心思想是：先用激光跟踪仪靶标点恢复各工装端面的空间坐标系，再把测量场映射到设备理论基准场，最后把对接目标分解为地轨平移、4 条顶升支链高度和固定段滚转补偿指令。

生成日期：2026-07-07。当前 laser_tracker_data.mat 在 Git 工作树中处于已修改状态；本文数值结果基于当前文件内容复算。

单位：mm / degree 靶标数：18 个 当前模式：align_mode = 3 数据噪声：noise_level = 0

一、总体结论

程序用途

它不是单纯画图脚本，而是一个“测量点生成 + 靶标坐标系拟合 + 设备基准映射 + 对接逆解”的完整原型。

数学主线

全部建模建立在齐次变换矩阵、三点定坐标系、三点定圆心、轴线投影正交化、圆柱接触几何求根这些基础上。

当前求解结果

人工调整模式下，地轨 Y 向平移约 +197.006 mm；4 条支链目标高度为 1494.306 / 1499.283 / 1706.188 / 1610.398 mm。

1. 生成靶标按世界、基座、固定段、移动段四类点构造 18 个点。
2. 读取测点从 laser_tracker_data.mat 读取 Targets。
3. 拟合坐标系由靶标点恢复 W、F、M0 的位姿。
4. 空间映射把激光跟踪仪测量场转到设备理论场。
5. 对接策略选择完全重合、姿态平行或人工 6-DOF 调整。
6. IK 分解输出固定段 Roll、移动段 Y 平移和 4 支链高度。

齐次变换约定

脚本使用列向量点坐标和 4x4 齐次矩阵：

T = [ R  p ]
    [ 0  1 ]

p_global = R * p_local + p
[p_global; 1] = T * [p_local; 1]

gen_markerdata_V3.m 中的 T_M0_base * Delta_M0、T_F_base * Delta_F 用来模拟实际安装或停放误差。 simulation_mapping_ik_v1.m 中的 T_M0 * T_manual 则表示人工调整量作用在移动段当前局部坐标系下，这是理解 man_dx/man_dy/man_dz 的关键。

坐标系含义

坐标系来源用途
{W}P1-P3 三个世界系靶标定义设备理论基准场中的世界框架，理想原点位于 Y = Lw/2。
{B}P4-P6 三个基座点生成器中用于辅助标定地基/基座关系，主求解脚本目前没有直接使用 B 的拟合结果。
{F}P7-P12 两组固定段截面圆周点拟合固定段目标端面位姿，作为对接目标或姿态参照。
{M0}P13-P18 两组移动段截面圆周点拟合移动段当前端面位姿，作为逆解初始状态。
{Mnew}由策略生成移动段需要到达的目标位姿，用于 IK 分解。

四、靶标点设计与数据生成

gen_markerdata_V3.m 的作用是构造一套可验证算法的“激光跟踪仪靶标点云”。它先定义理论几何尺寸，再人为注入移动段和固定段的 6-DOF 微调误差，最后把各局部坐标系中的靶标点转换到全局测量坐标中。

点号归属生成方式算法用途
P1-P3世界系 {W}三点定义原点、Y 方向和辅助平面点。恢复测量场中的世界坐标系。
P4-P6基座系 {B}三点定义基座参考。当前主程序未参与求解，可作为扩展校核。
P7-P12固定段 {F}两截面，每截面 3 个圆周点；局部 y = 500 和 1500。三点定圆，得到两个截面圆心，恢复固定段轴线和端面。
P13-P18移动段 {M0}两截面，每截面 3 个圆周点；局部 y = -500 和 -1500。同上，用于恢复移动段当前位姿。

生成器中注入的模拟误差为：

对象平移误差 [dx, dy, dz]转角误差 [rx, ry, rz]
移动段 {M0}[15, -25, 12] mm[0.5, 0.0, -0.8] deg
固定段 {F}[150, 0, 100] mm[1.5, 3.0, 0.5] deg
当前 noise_level = 0，因此数据是理想、无噪声的。用这份数据验证算法时，拟合结果应当几乎精确恢复注入的位姿误差。

五、坐标系拟合算法

1. 世界系三点拟合

主程序使用 P1、P2、P3 构造世界坐标系：

O_W = P2
Y_W = normalize(P1 - P2)
Z_W = normalize((P3 - P2) x (P1 - P2))
X_W = normalize(Y_W x Z_W)
T_W_fitted = [X_W, Y_W, Z_W, O_W; 0 0 0 1]

这个构造依赖点的顺序。若 P1/P2/P3 交换，可能导致轴向反号或坐标系变成错误的手性。

2. 三点求空间圆心

固定段和移动段每个截面用 3 个圆周点求圆心。代码中的解析公式是：

v1 = p2 - p1
v2 = p3 - p1
n  = v1 x v2
C  = p1 + ((||v1||^2 v2 - ||v2||^2 v1) x n) / (2 ||n||^2)

得到近端圆心 C1 和远端圆心 C2 后，轴线方向由两圆心连线确定。 固定段使用 Y_F = normalize(C2_F - C1_F)，移动段使用 Y_M = normalize(C1_M - C2_M)，这是由两组靶标在局部 y 方向的布置决定的。

3. X/Z 轴恢复与正交化

脚本选取截面中角度为 0 度附近的靶标点作为 X 轴方向参考，再去掉它在 Y 轴上的投影：

X_temp = normalize(P_ref - C2)
X      = normalize(X_temp - dot(X_temp, Y) * Y)
Z      = X x Y

这样可以避免截面点因微小误差不完全垂直于轴线时造成坐标轴不正交。

六、测量场到设备场的空间映射

激光跟踪仪可以任意放置，因此直接测出的 Targets 属于测量场。程序先在测量场中拟合出 T_W_fitted，再用理论世界系 T_W_ideal 构造转换： 

T_LT2Machine = T_W_ideal * inv(T_W_fitted)
T_M0         = T_LT2Machine * T_M0_fitted
T_F_curr     = T_LT2Machine * T_F_fitted
Targets_Machine(i) = T_LT2Machine * Targets(i)

含义是：把所有实测位姿从“激光跟踪仪坐标”统一映射回“设备理论基准坐标”。这一步是后续 IK 能成立的前提。 当前数据中 P1-P3 与理论世界系完全一致，因此复算得到 T_LT2Machine 基本为单位变换。

七、对接策略与逆运动学

对接策略

模式代码行为适用理解
align_mode = 1T_Mnew = T_F_curr移动段端面与固定段端面完全重合。
align_mode = 2姿态取固定段，位置保留移动段当前值。只要求姿态平行，不做端面位置对齐。
align_mode = 3T_Mnew = T_M0 * T_manual人工输入 6-DOF 调整量；当前代码使用 [150, 200, 100] mm 和 [0, 0, 0] deg。

IK 分解逻辑

逆解将目标移动段圆柱轴线表示为：

P_a = T_Mnew(1:3, 4)     // 目标端面圆心
v_a = T_Mnew(1:3, 2)     // 目标圆柱轴线方向，即局部 Y 轴

移动段并联机构被分解为 5 个自由度控制量：地轨基座 Y 向平移 + 4 条支链顶升高度。

dy_cmd = T_Mnew_y - T_M0_y
Y_cols_new(i) = Y_cols(i) + dy_cmd

每条支链在自己的 Y 截面上与目标圆柱轴线相交：

t_i      = (Y_cols_new(i) - P_a_y) / v_a_y
X_axis_i = P_a_x + t_i * v_a_x
Z_axis_i = P_a_z + t_i * v_a_z

在 X-Z 截面中，滚筒中心到工件轴线距离等于 Rw + rr。若支链横向位置为 X_cols(i)，顶升高度为 H_i，则：

(X_cols(i) - X_axis_i)^2 + (H_i - Z_axis_i)^2 = (Rw + rr)^2
H_i = Z_axis_i - sqrt((Rw + rr)^2 - (X_cols(i) - X_axis_i)^2)

代码选择负号，表示滚筒中心位于工件轴线下方。若根号内为负，说明该支链在当前横向位置下已经无法接触目标圆柱，程序会报“幅度超限/脱轨”。

固定段 Roll 补偿

Delta_T_M       = T_M0 \ T_Mnew
target_roll_M   = rad2deg(asin(Delta_T_M(1,3)))
roll_cmd_fixed  = -target_roll_M

这相当于从移动段相对变化中取一个滚转分量，并让固定段工装做相反方向补偿。当前人工模式没有引入旋转，所以补偿结果约为 0 度。

九、你需要特别注意的事项

1. 当前算法对噪声不鲁棒。 每个圆截面只用 3 个点精确求圆心。如果真实激光跟踪仪数据有噪声、遮挡或离群点，建议改为多点最小二乘圆/圆柱拟合，并加入残差评估。
2. 三点退化没有保护。 fitCircle3D 中若三点近似共线，则 norm(n)^2 很小，会导致数值爆炸。世界系三点若共线，也无法稳定构造坐标系。
3. v_a(2) 接近 0 时 IK 会失效。 公式 t_i = (Y_cols_new(i) - P_a(2)) / v_a(2) 默认目标轴线有明显 Y 分量。如果目标圆柱轴线接近横向，会出现除零或高度发散。
4. 可达性只检查了横向根号。 代码用 inner_val < 0 判断支链横向脱轨，但没有检查顶升高度上下限、地轨行程、滚筒姿态、碰撞、同步误差和速度/加速度约束。
5. 欧拉角显示仅供本程序约定。 printPose 用 asin/atan2 从矩阵取 Rx/Ry/Rz；这组角度需要和控制器采用的旋转顺序一致，不能直接假定等同于任意软件中的欧拉角定义。
6. 控制策略目前是硬编码。 align_mode = 3、人工调整量 [150, 200, 100, 0, 0, 0] 都写死在脚本中。若要工程使用，应改成参数输入、配置文件或 GUI 控件。
7. 中文文件内容本身是 UTF-8，部分终端可能显示乱码。 用按字节/UTF-8 方式复核后，CSV 导出循环在第 100 行独立存在，代码结构正常；PowerShell 默认编码显示可能把中文注释显示成乱码。

十、建议的工程化改进

优先级建议原因
高 把 fitCircle3D 升级为多点最小二乘圆/圆柱拟合，并输出 RMS 残差。 真实测量数据通常不满足三点无噪声假设，残差可作为靶标质量和装夹异常的判据。
高 为 v_a(2)、norm(n)、inner_val、顶升高度和地轨行程加入阈值检查。 把数学不可解、机构不可达和测量退化区分开，便于现场排障。
中 把 T_W_ideal * inv(T_W_fitted) 改为更稳定的刚体变换求逆函数，或 MATLAB 右除 T_W_ideal / T_W_fitted。 避免显式求逆带来的数值和风格问题；SE(3) 矩阵可用 R' 和 -R' p 快速求逆。
中 把策略模式、人工调整量、机构限位、输入输出文件路径参数化。 减少脚本硬编码，便于批处理不同工况。
中 增加“无噪声回归测试”：生成器输出后，主程序应恢复注入的 M0/F 位姿，并输出可比较误差。 保证后续改算法时不会破坏坐标系方向、点号约定和 IK 符号。
低 把 P4-P6 基座系纳入校核或删去未使用逻辑。 当前主程序读入但不使用基座三点，容易让维护者误解其作用。

十一、一句话总结

这套 MATLAB 程序建立了一个清晰的空间对接原型：用靶标点恢复端面位姿，用世界系标定消除激光跟踪仪摆放误差，用目标圆柱轴线与支链截面几何关系求出 4-PR 顶升高度。 它适合作为仿真验证和算法雏形；若要走向真实设备控制，关键补强点是测量鲁棒性、机构限位、异常诊断、参数化和控制器角度约定的一致性。
