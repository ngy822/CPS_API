2\. 对测量结果影响最大的现场条件

最关键的是靶点顺序。当前解算器至少要求 18 个有效点：

第 1-3 点：世界/基准坐标系拟合点

第 7-12 点：固定段端面靶点，两组三点拟合圆

第 13-18 点：移动段端面靶点，两组三点拟合圆

代码中第 4-6 点目前没有参与解算，但 CSV 仍要求整体顺序保持模板一致。不要现场临时改点序，否则位姿会算偏。

CSV 格式是：

Index,X,Y,Z,TargetSize,Timestamp

坐标单位是 mm。

**4. 工件或支撑机构变化时必须改的几何参数**

这些在 \[AlignmentSolver.cpp (line 31)](G:/CPS\_API/SenpecKinematicsPoseTracker/src/core/AlignmentSolver.cpp:31) 里固定：

**Rw = 3506.0 / 2.0;   // 筒体半径**

**Lw = 9580.0;         // 筒体长度**

**Xc = 2710.0 / 2.0;   // 支撑/轮组 X 向半距**

**Yc = 4500.0 / 2.0;   // 支撑/轮组 Y 向半距**

**rr = 850.0 / 2.0;    // 滚轮半径**

**Rw：筒体有多粗**

**Lw：筒体有多长**

**Xc：左右支撑有多宽**

**Yc：前后支撑有多远**

**rr：滚轮有多大**

**还有两个重要的端面偏移量写死为 500.0 mm：**

**O\_F\_f = C1\_F - Y\_F\_f \* 500.0;**

**O\_M\_f = C1\_M + Y\_M\_f \* 500.0;**

**如果现场靶标环到端面的距离不是 500 mm，这个一定要改，否则目标端面坐标系会整体偏移**。

5\. 对接策略参数

现在界面调用的是：

solver\_.computeAlignment(kinematicsCsvPath\_, 1);

也就是固定使用“模式 1：完全重合”。

解算器其实支持三种模式：

模式 1：移动段目标位姿完全对齐固定段端面

模式 2：只让姿态平行，位置保持移动段当前中心

模式 3：人工输入 dx/dy/dz/rx/ry/rz 微调

但目前 UI 没有暴露模式选择和手动微调参数。现场如果需要“保留安全间隙”“只先调姿态不推进”“人工补偿几毫米”，建议下一步把 align\_mode 和 man\_dx/man\_dy/man\_dz/man\_rx/man\_ry/man\_rz 做成界面参数。

6\. CPS/UE5 联动参数

在 \[DigitalTwinPublisher.cpp (line 22)](G:/CPS\_API/SenpecKinematicsPoseTracker/src/core/DigitalTwinPublisher.cpp:22)：

kBusServerIp = "127.0.0.1";

kBusServerPort = 5000;

kLogServerIp = "127.0.0.1";

kLogServerPort = 8888;

kPublishInterval = 50 ms;

如果 CPS 总线和 UE5 在同一台电脑，保持 127.0.0.1 即可。

如果 CPS 总线服务或 UE5 接收端换到别的机器，需要改 IP/端口。

如果 UE5 显示卡顿或网络负载高，可以适当增大发布周期，比如 100 ms。

现场优先检查顺序

我建议你现场按这个顺序检查：

跟踪仪 IP 是否能连上。

靶标球尺寸是否和实物一致。

光路锁定、信号强度、预热状态是否正常。

测点数量和顺序是否符合模板。

RMS 是否过大，必要时延长单点平均时间。

工件尺寸、靶标到端面距离、支撑轮距是否与源码参数一致。

对接策略是否真的是“完全重合”，如果不是，需要开放模式 2/3。
