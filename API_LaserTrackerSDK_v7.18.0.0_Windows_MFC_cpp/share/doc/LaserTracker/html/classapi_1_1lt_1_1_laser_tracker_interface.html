◆ Connect() [1/3]

api::lt::ErrorType api::lt::LaserTrackerInterface::Connect ( const char *  ip_address,
const char *  camera_ip_address,
TaskCallback  callback_function,
void *  callback_parameter = nullptr,
const char *  prm_files_path = nullptr,
const char *  dll_directory = nullptr 
)

Function to connect to the hardware device.

This should be the first function to be called in order to setup the connection to hardware. On calling this function, it will evaluate the input parameters and return to the caller either Success or an appropriate error code. A task with the id TaskMode::Connect is assigned. Upon the task completion (whether success or failure) a notification callback is issued. With the results of the callback, applications can know the status of the connection.

Parameters
[in]ip_addressEthernet ip address for the device.
[in]camera_ip_addressEthernet ip address for the camera submodule.
[in]callback_functionThis is the function pointer to get the notification of the task completion. For the list of tasks please see, api::lt::TaskMode enumeration for more details.
[in]callback_parameterA pointer to data used by the callback. Default is nullptr.
[in]prm_files_pathDirectory where the prm files for the device has to be stored. If nullptr then the current directory is chosen. Default is nullptr
[in]dll_directoryDirectory where the platforms directory of Qt library will be placed. nullptr then the application directory is chosen. Default is nullptr
Returns
ErrorType::Success In case of success
ErrorType::DeviceAlreadyConnected In case device already connected and trying to connect again
ErrorType::InvalidCallbackParameter In case callback_function pointer is invalid

◆ FlipFrontSightBackSight()

api::lt::ErrorType api::lt::LaserTrackerInterface::FlipFrontSightBackSight ( )

Function to flip the Laser tracker gimbal between frontsight and backsight.

The gimbal will be flipped to other side i.e., $$ az_{target} = 180 \pm az_{current} $$ $$ el_{target} = 180 - el_{current} $$

Note
This function call will only flip the gimbal. It will not perform any measurements If the function returns ErrorType::Success, TaskMode::FlipFSBS is assigned and a callback is sent when the task completes.
Returns
ErrorType::Success If the task is successfully started.
ErrorType::DeviceNotConnected If device is not connected.
ErrorType::OtherTaskOnGoing If the device is performing another task.
